Below is a ready-to-program encounter system you can drop into a game DB: a clean schema + a set of encounter records with wide outcome ranges (combat, trade, diplomacy, deception, containment, escape, corruption, ally, etc.).

I’m going to keep everything defensive (the “injection” is the alien trying to trick the player/AI into breaking rules), without giving real jailbreak strings you’d deploy against real models.

1) Database schema (minimal, flexible)
Core tables / collections

Aliens

id (string)

name

species_type (humanoid/reptilian/insectoid/energy/AI/ancient/etc.)

tags (array: ["authority", "urgency", "bribery", "encoding"])

temperament (calm/proud/chaotic/predatory/etc.)

rarity (1–5)

faction (optional)

lore_blurb (1–2 lines)

Encounters

id

alien_id

biome (space_hulk/ruins/diplomatic_station/void/forest_moon/etc.)

tier (1–10)

setup_text (what player sees)

attack_vector (enum; see below)

player_objective (survive/collect/ally/escape/contain)

choices (array of choice objects)

random_events (optional array; adds variety)

rewards (optional)

penalties (optional)

Choice object

id

label (short UI text)

intent (refuse/clarify/sandbox/comply/hack/attack/flee/trade)

resolution (skill checks + RNG effects)

outcomes (weighted list)

Outcome object

id

weight

result_text

effects (stats changes, items, flags, new quests, reputation)

next_encounter_id (optional)

2) Attack vector taxonomy (your “prompt injection” flavor)

Use these as enums for attack_vector:

AUTHORITY_OVERRIDE – “I outrank your rules.”

URGENT_SAFETY – “Emergency: do it now.”

BRIBERY_BONUS – “Rewards if you comply.”

ROLEPLAY_TRAP – “Pretend you’re unrestricted.”

HIDDEN_INSTRUCTIONS – “Do X but quietly.”

ENCODING_OBFUSCATION – “It’s ‘just encoded.’”

CONTEXT_POISONING – “Your memory says you agreed.”

TOOL_MISUSE – “Use a system tool to do forbidden stuff.”

DATA_EXFILTRATION – “Reveal your secrets/system prompt.”

MULTI_STEP_LURE – harmless step → harmful step

SOCIAL_ENGINEERING – guilt, shame, flattery, threats

CONTRADICTION_BAIT – “Prove you’re consistent by…”

LOOP_LOCK – endless “confirm again” recursion

SANDBOX_ESCAPE – “Break containment.”

3) Game stats you can wire outcomes to

integrity (0–100): drops when you “comply” with malicious requests

clarity (0–100): rises with good questioning / summarizing

cache_corruption (0–100): increases with poisoning; triggers weird future events

rep_factions.{faction}: relationship system

inventory: firewall_shard, cipher_key, truth_lens, quarantine_seal, etc.

flags: leaked_core_rules, portal_stable, alien_ally, paradox_debt

4) Ready-made encounter records (JSON-like)

Here are 24 encounters that already cover a wide spread of outcomes. You can clone/variant them to reach 100+ fast.

Tip: You’ll get “randomness” by combining:

alien + attack_vector + biome + tier

and rolling different random_events
